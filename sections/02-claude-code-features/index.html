    Claude Code 사용법

    Section 02 · Lecture + Demo · 20분

    이번 섹션에서는 Claude Code의 설치부터 핵심 기능, 실용 팁까지 전반적인 사용법을 다룹니다. Section 01에서 배운 AI 기반 개발의 개념을 실제 도구로 체험하는 시간입니다.

    학습 목표

    • Claude Code CLI의 설치와 기본 사용법을 숙지한다
    • CLAUDE.md를 활용한 프로젝트 컨텍스트 관리법을 이해한다
    • 주요 명령어와 실용 팁을 학습한다
    • 실제 데모를 통해 Claude Code의 워크플로우를 체험한다
    학습 목표를 하나씩 짚어봅니다. 이 섹션이 끝나면 여러분은 Claude Code를 직접 설치하고, 터미널에서 자연어로 코드를 작성할 수 있게 됩니다. CLAUDE.md 파일을 통한 컨텍스트 관리도 핵심 내용입니다.

    Claude Code란?

    CLI 기반 AI 코딩 어시스턴트

    • 터미널에서 직접 실행하는 에이전틱 코딩 도구
    • 자연어로 코드 생성, 수정, 디버깅
    • 파일 시스템, 터미널, 웹에 직접 접근
    • 기존 개발 환경을 바꾸지 않고 사용
    개발자 (You)
    "함수 만들어줘"
    ↓ 자연어
    Claude Code
    (CLI Agent)
    ↓
    파일
    R/W
    ↓
    터미널
    실행
    ↓
    웹
    검색
    Claude Code는 Anthropic이 만든 CLI 기반 AI 코딩 어시스턴트입니다. IDE 플러그인이 아니라 터미널에서 직접 실행하는 도구입니다. 에이전틱이라는 말은, 사용자의 요청을 받아 스스로 파일을 읽고 쓰고, 터미널 명령을 실행하고, 필요하면 웹 검색까지 해서 작업을 완수한다는 의미입니다.

    설치 방법

    1단계: Node.js 확인

    # Node.js 18+ 필요
node --version
# v20.x.x 이상 권장

    2단계: Claude Code 설치

    # npm으로 전역 설치
npm install -g @anthropic-ai/claude-code

    3단계: 인증

    # 최초 실행 시 브라우저에서 Anthropic 계정 인증
claude
    Tip: 설치 후 claude --version으로 정상 설치 확인
    설치는 매우 간단합니다. Node.js 18 이상이 필요하고, npm으로 전역 설치합니다. 최초 실행 시 브라우저가 열리면서 Anthropic 계정 인증을 진행합니다. 이미 설치된 분들은 --version으로 확인해보세요.

    Endpoint 설정 (1/3)

    Step 1. Databricks에서 모델 선택

    Databricks Workspace에서 Coding Agent Integration 설정

    Claude Code를 선택하고 사용할 모델(Sonnet, Opus 등)을 지정합니다.
    설치가 끝나면 Databricks를 통해 endpoint를 설정합니다. Databricks Workspace에서 Coding Agent Integration 메뉴로 들어가면 Claude Code를 선택할 수 있고, 기본 모델과 Opus 모델을 각각 지정할 수 있습니다.

    Endpoint 설정 (2/3)

    Step 2. settings.json 설정

    ~/.claude/settings.json에 Databricks endpoint 정보를 입력합니다.

    Generate API Key 버튼으로 토큰을 생성하고 설정에 반영합니다.
    Databricks에서 자동으로 생성해주는 settings.json 설정을 복사합니다. ANTHROPIC_BASE_URL에 Databricks serving endpoint URL이 들어가고, ANTHROPIC_AUTH_TOKEN에는 Generate API Key로 생성한 토큰을 넣습니다. 이렇게 하면 Claude Code가 Databricks를 통해 모델에 접근하게 됩니다.

    Endpoint 설정 (3/3)

    Step 3. 프로젝트 레벨 설정

    프로젝트 루트의 .claude/ 디렉토리에서 권한을 관리합니다.

    .claude
└── settings.local.json

    settings.local.json 예시

    {
    "env": {
        "ANTHROPIC_MODEL": "databricks-claude-sonnet-4-5",
        "ANTHROPIC_BASE_URL": "https://e2-demo-field-eng.cloud.databricks.com/serving-endpoints/anthropic",
        "ANTHROPIC_AUTH_TOKEN": "your-databricks-token",
        "ANTHROPIC_DEFAULT_OPUS_MODEL": "databricks-claude-opus-4-6",
        "ANTHROPIC_CUSTOM_HEADERS": "x-databricks-use-coding-agent-mode: true",
        "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": "1"
    }
}
    Tip: settings.local.json은 개인 설정 파일로, .gitignore에 추가하여 개인별로 관리하는 것을 권장합니다.
    마지막으로 프로젝트 레벨에서 permissions을 설정합니다. 프로젝트 루트에 .claude 디렉토리를 만들고, settings.local.json 파일로 권한을 관리합니다. permissions의 allow 배열에 Claude Code가 사용할 수 있는 도구와 명령어를 정의합니다. 이 파일은 개인 설정이므로 .gitignore에 추가하여 팀원마다 각자 관리하는 것이 좋습니다.

    기본 사용법

    대화형 모드

    # 프로젝트 디렉토리에서 실행
claude

# Claude Code가 시작되면 자연어로 대화
> "이 프로젝트의 구조를 설명해줘"
> "tests/ 폴더에 유닛 테스트 만들어줘"
> "README.md 업데이트해줘"

    원샷 모드 (비대화형)

    # -p 플래그로 한 번에 실행
claude -p "setup.py의 버전을 1.2.0으로 변경해줘"

# 파이프와 함께 사용
cat error.log | claude -p "이 에러의 원인을 분석해줘"
    기본적으로 프로젝트 디렉토리에서 claude를 실행하면 대화형 모드로 들어갑니다. 자연어로 원하는 작업을 요청하면 됩니다. -p 플래그를 쓰면 원샷 모드로, 하나의 명령만 실행하고 종료합니다. 파이프와 함께 사용하면 로그 분석 같은 작업에도 활용할 수 있습니다.

    핵심 기능 (1/4)

    코드 생성 & 리팩토링

    코드 생성

    > "FastAPI로 사용자 CRUD API 만들어줘"

> "pandas로 CSV 파일 읽어서
   월별 매출 차트 그려주는 함수 만들어줘"

    리팩토링

    > "이 함수를 async로 변환해줘"

> "중복 코드를 공통 유틸리티로
   추출해줘"

> "타입 힌트 추가해줘"
    Claude Code는 프로젝트의 기존 코드 스타일과 패턴을 파악하여 일관된 코드를 생성합니다.
    첫 번째 핵심 기능은 코드 생성과 리팩토링입니다. 단순히 코드를 생성하는 것이 아니라, 프로젝트의 기존 코드를 읽고 스타일과 패턴을 파악한 뒤 그에 맞춰 일관된 코드를 생성합니다. 리팩토링도 마찬가지로, 자연어로 요청하면 코드를 분석하고 개선해줍니다.

    핵심 기능 (2/4)

    파일 읽기 / 쓰기 / 편집

    # 파일 읽기 - 프로젝트 전체를 자동으로 탐색
> "src/main.py 파일의 내용을 설명해줘"

# 파일 생성
> "config/settings.yaml 파일 만들어줘"

# 파일 편집 - 정확한 위치에 변경 적용
> "app.py의 23번째 줄에 있는 버그 수정해줘"

# 다중 파일 편집
> "모든 테스트 파일에서 deprecated된 assert 메서드를
   새로운 방식으로 바꿔줘"
    안전장치: 파일 수정 전 항상 확인을 요청합니다. --allowedTools로 권한을 세밀하게 제어할 수 있습니다.
    두 번째 핵심 기능은 파일 조작입니다. Claude Code는 프로젝트의 파일 구조를 자동으로 탐색하고, 파일을 읽고, 새로 만들고, 편집할 수 있습니다. 중요한 점은 파일을 수정하기 전에 항상 사용자에게 확인을 요청한다는 것입니다. 실수로 중요한 파일을 덮어쓸 걱정이 없습니다.

    세 번째 핵심 기능은 터미널 명령 실행입니다. 테스트 실행, 패키지 설치, Git 작업 등을 자연어로 요청할 수 있습니다. 특히 "테스트 돌려보고 실패하면 수정해줘"처럼 조건부 작업도 가능합니다. 에이전틱하게 동작하기 때문에 테스트 실패 → 코드 수정 → 재실행 루프를 자동으로 수행합니다.

    웹 검색

# 라이브러리 사용법 확인

# 에러 해결

    Claude Code는 학습 데이터 이후의 최신 정보도 웹 검색을 통해 활용할 수 있습니다.
    네 번째 핵심 기능은 웹 검색입니다. Claude의 학습 데이터에 없는 최신 라이브러리 문서나, 에러 메시지 해결법을 실시간으로 검색할 수 있습니다. 이는 특히 빠르게 변하는 프레임워크를 사용할 때 매우 유용합니다.

    CLAUDE.md

    프로젝트 컨텍스트 파일

    CLAUDE.md란?

    • 프로젝트 루트에 위치하는 컨텍스트 파일
    • Claude Code가 세션 시작 시 자동으로 읽음
    • 프로젝트의 규칙, 구조, 스타일 가이드 정의
    • 팀원 간 AI 사용 일관성 보장
    # CLAUDE.md

## 프로젝트 개요
Python 3.11 기반 FastAPI 백엔드

## 코딩 규칙
- Black 포매터 사용 (line-length=88)
- 모든 함수에 타입 힌트 필수
- docstring은 Google 스타일

## 테스트
- pytest 사용
- tests/ 디렉토리에 위치
- 커버리지 80% 이상 유지

## 주의사항
- .env 파일은 절대 커밋하지 않음
- API 키는 환경변수로 관리
    CLAUDE.md는 프로젝트의 "설명서"와 같습니다. 프로젝트 루트에 이 파일을 두면, Claude Code가 세션을 시작할 때 자동으로 읽습니다. 프로젝트의 기술 스택, 코딩 규칙, 테스트 방식 등을 정의하면 Claude Code가 더 정확하고 일관된 코드를 생성합니다. 팀 전체가 같은 CLAUDE.md를 공유하면 AI 사용의 일관성도 보장됩니다.

    CLAUDE.md 작성 팁

    항목 설명 예시
    프로젝트 개요 기술 스택, 목적 Python 3.11 + FastAPI
    코딩 규칙 포매터, 린터 설정 Black, Ruff 사용
    디렉토리 구조 주요 폴더 역할 src/ - 소스, tests/ - 테스트
    빌드/실행 명령 자주 쓰는 명령 make dev, make test
    주의사항 하지 말아야 할 것 .env 커밋 금지
    핵심: 간결하고 명확하게. 새로 합류한 팀원에게 설명하듯 작성하세요.
    CLAUDE.md를 잘 작성하는 팁입니다. 핵심은 "새로 합류한 팀원에게 설명하듯 작성하라"는 것입니다. 너무 길면 오히려 효과가 떨어지니, 핵심 정보만 간결하게 담으세요. 프로젝트 개요, 코딩 규칙, 디렉토리 구조, 빌드 명령, 주의사항 이 5가지가 필수입니다.

    컨텍스트 관리

    파일 참조 방법

    # 파일 경로를 직접 언급
> "src/utils/helpers.py를 보고 비슷한 패턴으로 새 유틸 함수 만들어줘"

# 디렉토리 전체 참조
> "tests/ 폴더의 기존 테스트 패턴을 따라서 새 테스트 작성해줘"

# 여러 파일 동시 참조
> "models.py와 schemas.py를 참고해서 새로운 엔드포인트 추가해줘"

    컨텍스트 우선순위

    1. CLAUDE.md (자동 로드)
2. 현재 대화의 이전 메시지
3. 명시적으로 참조한 파일
4. Claude가 자동 탐색한 파일
    컨텍스트 관리는 Claude Code를 효과적으로 사용하는 핵심입니다. 파일 경로를 직접 언급하면 Claude Code가 해당 파일을 읽고 참조합니다. 컨텍스트는 우선순위가 있습니다: CLAUDE.md가 가장 높고, 그 다음이 대화 이력, 명시적으로 참조한 파일, 그리고 Claude가 자동으로 탐색한 파일 순입니다.

    실용 팁

    주요 명령어

    명령어 기능 사용 시기
    /help 도움말 표시 사용법이 궁금할 때
    /clear 대화 기록 초기화 새로운 작업 시작 시
    /compact 대화 요약 후 압축 컨텍스트가 길어졌을 때
    Shift+Tab Plan Mode 전환 실행 전 계획을 먼저 확인
    Esc 현재 작업 중단 잘못된 방향으로 갈 때
    실용 팁으로 자주 쓰는 명령어를 정리했습니다. 특히 /compact는 대화가 길어져서 컨텍스트 윈도우가 부족해질 때 매우 유용합니다. 대화를 요약해서 압축하므로 토큰을 절약할 수 있습니다. Shift+Tab으로 Plan Mode에 진입하면, 코드를 바로 수정하지 않고 계획만 먼저 보여줍니다. 복잡한 작업 전에 반드시 Plan Mode를 활용하세요.

    효과적인 프롬프트 작성법

    나쁜 예시

    > "코드 고쳐줘"

> "더 좋게 만들어줘"

> "에러 나는데 해결해줘"

    좋은 예시

    > "get_user 함수에서 DB 연결이
   실패할 때 retry 로직 추가해줘.
   최대 3회, 간격은 exponential
   backoff으로."

> "src/api/routes.py의 /users
   엔드포인트에 pagination 추가해줘.

    원칙: 구체적인 파일, 함수, 요구사항을 명시할수록 좋은 결과를 얻습니다.
    프롬프트를 어떻게 작성하느냐에 따라 결과 품질이 크게 달라집니다. 왼쪽의 나쁜 예시처럼 모호하게 요청하면 Claude Code가 의도를 추측해야 합니다. 오른쪽의 좋은 예시처럼 파일명, 함수명, 구체적인 요구사항을 명시하면 훨씬 정확한 결과를 얻을 수 있습니다.

    데모

    Claude Code로 Python 함수 만들기

    시나리오: Claude Code를 사용해 간단한 Python 유틸리티 함수를 만들고 테스트까지 작성

    # 1. 프로젝트 초기화

# 4. 실행 및 확인
> "테스트 돌려줘"
    이제 실제 데모를 진행합니다. 간단한 프로젝트를 만들고, Claude Code에 fibonacci 함수를 요청합니다. 그 다음 테스트도 작성하게 하고, 실행까지 시켜봅니다. 이 전체 과정이 자연어로만 이루어진다는 점에 주목해주세요. 실제로 라이브 데모를 진행하면서 보여드리겠습니다.

    데모 결과 (예시)

    생성된 코드

    def fibonacci(n: int) -> list[int]:
    """Generate fibonacci sequence.

    Args:
        n: Number of fibonacci
           numbers to generate.

    Returns:
        List of n fibonacci numbers.

    Raises:
        ValueError: If n is negative.
    """
    if n < 0:
        raise ValueError(
            "n must be non-negative"
        )
    if n == 0:
        return []
    if n == 1:
        return [0]

    fib = [0, 1]
    for _ in range(2, n):
        fib.append(fib[-1] + fib[-2])
    return fib

    생성된 테스트

    import pytest
from fibonacci import fibonacci


def test_fibonacci_basic():
    assert fibonacci(5) == [
        0, 1, 1, 2, 3
    ]

def test_fibonacci_zero():
    assert fibonacci(0) == []

def test_fibonacci_one():
    assert fibonacci(1) == [0]

def test_fibonacci_negative():
    with pytest.raises(ValueError):
        fibonacci(-1)

def test_fibonacci_large():
    result = fibonacci(10)
    assert len(result) == 10
    assert result[-1] == 34
    이것이 Claude Code가 생성한 코드와 테스트의 예시입니다. 코드에 타입 힌트, Google 스타일 docstring, 에러 처리가 포함되어 있고, 테스트에는 기본 케이스뿐 아니라 edge case(0, 1, 음수)도 포함되어 있습니다. 이 모든 것이 자연어 한두 문장으로 생성되었습니다.
