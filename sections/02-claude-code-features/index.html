    Claude Code 사용법

    Section 02 · Lecture + Hands-on · 25분

    이번 섹션에서는 Claude Code의 설치부터 핵심 기능, 실용 팁까지 전반적인 사용법을 다룹니다. Section 01에서 배운 AI 기반 개발의 개념을 실제 도구로 체험하는 시간입니다.

    학습 목표

    • Claude Code CLI의 설치와 기본 사용법을 숙지한다
    • CLAUDE.md를 활용한 프로젝트 컨텍스트 관리법을 이해한다
    • 주요 명령어와 실용 팁을 학습한다
    • Claude Code로 Python 함수와 테스트를 직접 만들어본다
    학습 목표를 하나씩 짚어봅니다. 이 섹션이 끝나면 여러분은 Claude Code를 직접 설치하고, 터미널에서 자연어로 코드를 작성할 수 있게 됩니다. CLAUDE.md 파일을 통한 컨텍스트 관리도 핵심 내용입니다.

    Claude Code란?

    CLI 기반 AI 코딩 어시스턴트

    • 터미널에서 직접 실행하는 에이전틱 코딩 도구
    • 자연어로 코드 생성, 수정, 디버깅
    • 파일 시스템, 터미널, 웹에 직접 접근
    • 기존 개발 환경을 바꾸지 않고 사용

    Claude Code는 Anthropic이 만든 CLI 기반 AI 코딩 어시스턴트입니다. IDE 플러그인이 아니라 터미널에서 직접 실행하는 도구입니다. 에이전틱이라는 말은, 사용자의 요청을 받아 스스로 파일을 읽고 쓰고, 터미널 명령을 실행하고, 필요하면 웹 검색까지 해서 작업을 완수한다는 의미입니다.

    설치 방법

    1단계: Node.js 확인

    # Node.js 18+ 필요
node --version
# v20.x.x 이상 권장

    2단계: Claude Code 설치

    # npm으로 전역 설치
npm install -g @anthropic-ai/claude-code

    3단계: 인증

    # 최초 실행 시 브라우저에서 Anthropic 계정 인증
claude
    Tip: 설치 후 claude --version으로 정상 설치 확인
    설치는 매우 간단합니다. Node.js 18 이상이 필요하고, npm으로 전역 설치합니다. 최초 실행 시 브라우저가 열리면서 Anthropic 계정 인증을 진행합니다. 이미 설치된 분들은 --version으로 확인해보세요.

    Endpoint 설정 (1/3)

    Step 1. Databricks에서 모델 선택

    Databricks Workspace에서 Coding Agent Integration 설정

    Claude Code를 선택하고 사용할 모델(Sonnet, Opus 등)을 지정합니다.
    설치가 끝나면 Databricks를 통해 endpoint를 설정합니다. Databricks Workspace에서 Coding Agent Integration 메뉴로 들어가면 Claude Code를 선택할 수 있고, 기본 모델과 Opus 모델을 각각 지정할 수 있습니다.

    Endpoint 설정 (2/3)

    Step 2. settings.json 설정

    ~/.claude/settings.json에 Databricks endpoint 정보를 입력합니다.

    Generate API Key 버튼으로 토큰을 생성하고 설정에 반영합니다.
    Databricks에서 자동으로 생성해주는 settings.json 설정을 복사합니다. ANTHROPIC_BASE_URL에 Databricks serving endpoint URL이 들어가고, ANTHROPIC_AUTH_TOKEN에는 Generate API Key로 생성한 토큰을 넣습니다. 이렇게 하면 Claude Code가 Databricks를 통해 모델에 접근하게 됩니다.

    Endpoint 설정 (3/3)

    Step 3. 프로젝트 레벨 설정

    프로젝트 루트의 .claude/ 디렉토리에 설정 파일을 생성합니다.

    클립보드에서 바로 설정하기

    # 프로젝트 루트에서 실행
mkdir -p .claude && pbpaste > .claude/settings.local.json

# 설정 확인
cat .claude/settings.local.json

    settings.local.json 예시

    {
    "env": {
        "ANTHROPIC_MODEL": "databricks-claude-sonnet-4-5",
        "ANTHROPIC_BASE_URL": "https://e2-demo-field-eng.cloud.databricks.com/serving-endpoints/anthropic",
        "ANTHROPIC_AUTH_TOKEN": "your-databricks-token",
        "ANTHROPIC_DEFAULT_OPUS_MODEL": "databricks-claude-opus-4-6",
        "ANTHROPIC_CUSTOM_HEADERS": "x-databricks-use-coding-agent-mode: true",
        "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": "1"
    }
}
    Tip: 클립보드에 복사한 JSON을 위 명령어로 바로 적용할 수 있습니다. settings.local.json은 개인 설정 파일이므로 .gitignore에 추가하세요.
    수강생들은 사전에 제공받은 settings.local.json JSON을 클립보드에 복사해둔 상태입니다. 프로젝트 루트에서 mkdir -p .claude && pbpaste 명령어를 실행하면, 클립보드 내용이 바로 .claude/settings.local.json 파일로 저장됩니다. cat 명령어로 내용이 올바르게 들어갔는지 확인합니다. 이 파일은 개인 설정이므로 .gitignore에 추가하여 팀원마다 각자 관리하는 것이 좋습니다.

    대화형 모드

    기본적으로 프로젝트 디렉토리에서 claude를 실행하면 대화형 모드로 들어갑니다. 자연어로 원하는 작업을 요청하면 됩니다. -p 플래그를 쓰면 원샷 모드로, 하나의 명령만 실행하고 종료합니다. 파이프와 함께 사용하면 로그 분석 같은 작업에도 활용할 수 있습니다.

    CLAUDE.md

    프로젝트 컨텍스트 파일

    CLAUDE.md란?

    • 프로젝트 루트에 위치하는 컨텍스트 파일
    • Claude Code가 세션 시작 시 자동으로 읽음
    • 프로젝트의 규칙, 구조, 스타일 가이드 정의
    • 팀원 간 AI 사용 일관성 보장
    • /init 명령으로 자동 생성 가능
    # CLAUDE.md

## 프로젝트 개요
Python 3.11 기반 FastAPI 백엔드

## 코딩 규칙
- Black 포매터 사용 (line-length=88)
- 모든 함수에 타입 힌트 필수
- docstring은 Google 스타일

## 테스트
- pytest 사용
- tests/ 디렉토리에 위치
- 커버리지 80% 이상 유지

## 주의사항
- .env 파일은 절대 커밋하지 않음
- API 키는 환경변수로 관리
    CLAUDE.md는 프로젝트의 "설명서"와 같습니다. 프로젝트 루트에 이 파일을 두면, Claude Code가 세션을 시작할 때 자동으로 읽습니다. 프로젝트의 기술 스택, 코딩 규칙, 테스트 방식 등을 정의하면 Claude Code가 더 정확하고 일관된 코드를 생성합니다. 팀 전체가 같은 CLAUDE.md를 공유하면 AI 사용의 일관성도 보장됩니다.

    컨텍스트 관리

    파일 참조 방법

    # 파일 경로를 직접 언급
> "src/utils/helpers.py를 보고 비슷한 패턴으로 새 유틸 함수 만들어줘"

    @ (파일 경로 자동완성)

    # @를 입력하면 파일 경로 자동완성 활성화
> @src/utils/helpers.py 이 파일을 참고해서 새 유틸 만들어줘
> @tests/ 기존 테스트 패턴을 따라서 새 테스트 작성해줘
    @를 입력하면 파일 경로 자동완성이 활성화됩니다. 특정 파일을 빠르게 참조할 수 있습니다.
    컨텍스트 관리는 Claude Code를 효과적으로 사용하는 핵심입니다. 파일 경로를 직접 언급하면 Claude Code가 해당 파일을 읽고 참조합니다. 특히 @를 입력하면 파일 경로 자동완성이 활성화되어, 긴 경로를 직접 타이핑할 필요 없이 빠르게 파일을 참조할 수 있습니다.

    컨텍스트 우선순위

    1 CLAUDE.md — 자동 로드되는 프로젝트 설정
    2 현재 대화의 이전 메시지
    3 명시적으로 참조한 파일 (@로 지정)
    4 Claude가 자동 탐색한 파일
    컨텍스트에는 우선순위가 있습니다. CLAUDE.md가 가장 높은 우선순위를 가지며 항상 자동으로 로드됩니다. 그 다음이 현재 대화의 이전 메시지, 명시적으로 참조한 파일(@로 지정 가능), 그리고 마지막으로 Claude가 자동으로 탐색한 파일 순입니다. 우선순위가 높을수록 Claude의 응답에 더 큰 영향을 미칩니다.

    /compact 대화 요약 후 압축 컨텍스트가 길어졌을 때
    Shift+Tab Plan Mode 전환 실행 전 계획을 먼저 확인
    Esc 현재 작업 중단 잘못된 방향으로 갈 때
    ! Bash Mode (셸 명령 직접 실행) 명령 출력을 컨텍스트에 추가할 때
    @ 파일 경로 자동완성 특정 파일을 명시적으로 참조할 때
    실용 팁으로 자주 쓰는 명령어를 정리했습니다. /init은 프로젝트를 분석하여 CLAUDE.md 파일을 자동으로 생성해줍니다. 새 프로젝트 시작 시 가장 먼저 실행하면 좋습니다. 특히 /compact는 대화가 길어져서 컨텍스트 윈도우가 부족해질 때 매우 유용합니다. 대화를 요약해서 압축하므로 토큰을 절약할 수 있습니다. Shift+Tab으로 Plan Mode에 진입하면, 코드를 바로 수정하지 않고 계획만 먼저 보여줍니다. 복잡한 작업 전에 반드시 Plan Mode를 활용하세요. 느낌표(!)는 Bash Mode로, 셸 명령을 바로 실행하고 그 출력을 대화 컨텍스트에 추가합니다. 골뱅이(@)는 파일 경로 자동완성으로, 긴 경로를 직접 타이핑할 필요 없이 파일을 빠르게 참조할 수 있습니다.

    실습 결과 (예시)

    구현된 코드

    def fibonacci(n: int) -> list[int]:
    """Generate fibonacci sequence.

    Args:
        n: Number of fibonacci
           numbers to generate.

    Returns:
        List of n fibonacci numbers.

    Raises:
        ValueError: If n is negative.
    """
    if n < 0:
        raise ValueError(
            "n must be non-negative"
        )
    if n == 0:
        return []
    if n == 1:
        return [0]

    fib = [0, 1]
    for _ in range(2, n):
        fib.append(fib[-1] + fib[-2])
    return fib

    구현된 테스트

    import pytest
from exercise_01_fibonacci import (
    fibonacci,
)


def test_fibonacci_basic():
    assert fibonacci(5) == [
        0, 1, 1, 2, 3
    ]

def test_fibonacci_zero():
    assert fibonacci(0) == []

def test_fibonacci_one():
    assert fibonacci(1) == [0]

def test_fibonacci_negative():
    with pytest.raises(ValueError):
        fibonacci(-1)

def test_fibonacci_large():
    result = fibonacci(10)
    assert len(result) == 10
    assert result[-1] == 34
    이것이 Claude Code가 스켈레톤의 TODO를 구현한 결과 예시입니다. 코드에 타입 힌트, Google 스타일 docstring, 에러 처리가 포함되어 있고, 테스트에는 기본 케이스뿐 아니라 edge case(0, 1, 음수)도 포함되어 있습니다. 스켈레톤의 함수 시그니처와 TODO 주석만으로 Claude Code가 올바른 구현을 생성했습니다.
