    Vibe Coding이란?

    "

    There's a new kind of coding I call 'vibe coding' where you fully give in to the vibes, embrace exponentials, and forget that the code even exists.

    — Andrej Karpathy, 2025년 2월

    핵심 아이디어

    • 코드를 직접 작성하는 것이 아니라, AI에게 의도를 전달
    • 개발자의 역할: 코더 → 오케스트레이터
    • 자연어로 소프트웨어를 설계하고 구현하는 새로운 패러다임
    Andrej Karpathy는 Tesla의 AI 디렉터이자 OpenAI 공동 창업자로, 2025년 2월에 Vibe Coding이라는 용어를 처음 사용했습니다. 핵심은 개발자가 코드 한 줄 한 줄을 작성하는 것이 아니라, AI에게 무엇을 만들고 싶은지 자연어로 설명하고, AI가 코드를 생성하는 방식입니다. 오늘 핸즈온에서 이 경험을 직접 해보실 겁니다.

    AI 코딩 도구의 발전

    도구 방식 특징 한계
    GitHub Copilot 자동 완성 에디터 내 인라인 제안, Tab으로 수락 단일 파일 컨텍스트, 수동 적용
    Cursor 채팅 + 편집 멀티 파일 편집, Composer 모드 IDE 종속, 제한된 도구 실행
    Claude Code 에이전틱 터미널 기반, 자율적 도구 사용, MCP 확장 터미널 환경 필요

    자동 완성 → 대화형 편집 → 자율적 에이전트

    AI 코딩 도구의 발전 단계를 살펴봅시다. GitHub Copilot은 코드 자동 완성의 시작이었습니다. 에디터에서 Tab을 눌러 제안을 수락하는 방식이죠. Cursor는 한 단계 더 나아가서 채팅 기반으로 여러 파일을 동시에 편집할 수 있게 했습니다. 그리고 Claude Code는 에이전틱 방식으로, AI가 직접 파일을 읽고, 명령을 실행하고, 결과를 확인하며 자율적으로 개발 작업을 수행합니다. 이것이 바로 Vibe Coding을 가능하게 하는 핵심 기술입니다.

    Claude의 차별점

    긴 컨텍스트 윈도우

    200K 토큰 — 일반 코드베이스 전체를 한 번에 이해

    에이전틱 코딩

    파일 읽기/쓰기, 명령 실행, 검색까지 자율적으로 수행

    도구 사용 능력 (Tool Use)

    MCP를 통해 외부 서비스와 연동 — 무한 확장 가능

    뛰어난 코드 품질

    SWE-bench에서 최고 수준의 코드 생성 성능

    안전한 실행 환경

    사용자 승인 기반의 명령 실행 — 위험한 작업은 확인 요청

    자연스러운 한국어 지원

    한국어 프롬프트로 개발 가능, 번역 없이 직접 소통

    Claude가 다른 AI 코딩 도구와 구별되는 핵심 차별점입니다. 200K 토큰의 긴 컨텍스트 윈도우는 대규모 프로젝트의 코드를 한 번에 이해할 수 있다는 의미입니다. 에이전틱 코딩은 AI가 단순히 코드를 제안하는 것이 아니라, 직접 파일을 편집하고 명령을 실행한다는 점에서 혁신적입니다. MCP(Model Context Protocol)를 통한 도구 사용 능력은 오늘 핸즈온에서 직접 경험하실 핵심 기능입니다. 그리고 한국어를 자연스럽게 이해하므로, 한국어로 프롬프트를 작성해도 좋은 결과를 얻을 수 있습니다.

    에이전틱 코딩이란?

    AI가 도구를 사용하여 자율적으로 코드를 작성, 실행, 검증하는 방식

    💬 사용자 요청
    →
    🧠 계획 수립
    →
    🔧 도구 실행
    →
    🔍 결과 검증
    →
    ✅ 완료 보고

    기존 AI 코딩과의 차이

    기존 방식 (Copilot 등)

    • 사용자가 직접 코드 작성
    • AI는 제안만 제공
    • 사용자가 수동으로 적용

    에이전틱 방식 (Claude Code)

    • 사용자가 목표를 설명
    • AI가 계획 → 실행 → 검증
    • 파일 편집, 명령 실행까지 자동
    에이전틱 코딩의 핵심은 AI가 단순 제안자가 아니라 능동적인 실행자라는 점입니다. 사용자가 "FastAPI로 REST API를 만들어줘"라고 요청하면, Claude Code는 먼저 프로젝트 구조를 분석하고, 필요한 파일을 생성하고, 코드를 작성하고, 테스트까지 실행합니다. 기존 Copilot 방식은 한 줄씩 제안하는 반면, 에이전틱 방식은 전체 작업을 자율적으로 수행합니다. 물론 위험한 작업(파일 삭제, 시스템 명령)은 사용자 승인을 받습니다.

    질문이 있으신가요?

    세션을 마무리하며 핵심 내용을 정리합니다. Vibe Coding은 코드를 직접 작성하는 것이 아니라 AI에게 의도를 전달하는 새로운 패러다임이고, 에이전틱 코딩은 AI가 자율적으로 개발 작업을 수행하는 방식이며, Claude Code는 이를 가능하게 하는 강력한 도구라는 것을 기억해주세요. 질문을 받은 후 바로 다음 섹션으로 넘어가겠습니다. 다음 섹션에서는 실제로 Claude Code를 설치하고 사용해봅니다.
