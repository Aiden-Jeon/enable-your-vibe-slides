    AI Fluency란?

    "

    AI와 효과적으로, 효율적으로, 윤리적으로, 안전하게 협업할 수 있는 능력

    — Anthropic Academy, AI Fluency: Framework & Foundations

    • 도구 사용 이상의 사고방식 전환
    • AI 협업의 전체 생명주기를 체계적으로 관리
    • Anthropic이 제안하는 4D Framework로 구체화
    AI Fluency는 Anthropic Academy에서 제시하는 개념으로, AI 도구를 단순히 사용하는 것을 넘어 AI와 효과적으로 협업하는 역량을 의미합니다. 마치 외국어를 배울 때 단어만 아는 것과 유창하게 소통하는 것의 차이와 같습니다. 효과적이고, 효율적이고, 윤리적이고, 안전하게 — 이 네 가지 축을 모두 고려해야 합니다. 이를 체계적으로 실천하기 위한 프레임워크가 바로 4D Framework입니다.

    4D Framework

    Delegation (위임)

    AI에게 무엇을 맡기고, 무엇은 직접 할 것인가?

    Description (기술)

    AI에게 의도를 어떻게 명확히 전달할 것인가?

    Discernment (분별)

    AI의 결과물을 어떻게 평가할 것인가?

    Diligence (책임)

    AI를 윤리적이고 안전하게 사용하고 있는가?

    4D Framework는 AI 협업의 네 가지 핵심 축입니다. Delegation — 위임: 어떤 작업을 AI에게 맡길 것인지 판단하는 능력입니다. 모든 것을 맡기거나, 아무것도 맡기지 않는 것이 아니라 적절한 경계를 설정합니다. Description — 기술: AI에게 원하는 바를 명확히 전달하는 능력입니다. 프롬프트 엔지니어링이 여기에 해당합니다. Discernment — 분별: AI가 생성한 결과물의 품질을 평가하고 판단하는 능력입니다. Diligence — 책임: 보안, 윤리, 라이선스 등을 고려하여 AI를 책임감 있게 사용하는 것입니다. 오늘 핸즈온 전체에 걸쳐 이 네 가지를 계속 적용하게 됩니다.

    Delegation 깊이 보기

    누가 무엇을 할 것인가를 사려 깊게 결정하는 것

    나 혼자 • AI 혼자 • 함께 — 그리고 어떻게 분배할 것인가

    1. Problem Awareness

    문제 인식 —
    목표와 성공 기준을 먼저 명확히 한다

    2. Platform Awareness

    플랫폼 인식 —
    AI 시스템의 강점과 한계를 이해한다

    3. Task Delegation

    작업 분배 —
    사람 vs AI vs 협업, 전략적으로 배분한다

    4D 중 Delegation을 좀 더 깊이 살펴봅시다. Delegation은 단순히 "AI에게 시키자"가 아닙니다. 내가 할 일, AI가 할 일, 함께 할 일을 사려 깊게 결정하는 것입니다. 이를 위해 세 가지 기둥이 필요합니다. 첫째, Problem Awareness — 내가 풀려는 문제가 무엇인지 명확히 이해하는 것. 둘째, Platform Awareness — 사용 가능한 AI 도구의 강점과 한계를 파악하는 것. 셋째, Task Delegation — 이 이해를 바탕으로 작업을 전략적으로 분배하는 것. 다음 슬라이드에서 각 기둥을 좀 더 구체적으로 살펴보겠습니다.

    Delegation 3 Pillars

    1. Problem Awareness — 무엇을 하려는가?

    • "성공"은 어떤 모습인가? 어떤 사고와 작업이 필요한가?
    • AI fluency는 본인의 전문성에서 시작한다

    2. Platform Awareness — AI는 무엇을 잘하는가?

    • 어떤 모델이 적합한가? 속도 vs 창의성 vs 깊이 vs 정확성
    • 도구별 강점과 한계를 파악한다

    3. Task Delegation — 자, 이제 계획을 세우자

    • 자동화할 수 있는 것은? 협업이 더 가치 있는 곳은? 사람만 해야 할 것은?
    • 목표: 모든 것을 자동화하는 것이 아니라 최적의 인간-AI 파트너십
    세 기둥을 하나씩 보겠습니다. Problem Awareness는 AI를 사용하기 전에 내가 풀려는 문제를 명확히 이해하는 것입니다. 성공이 어떤 모습인지, 어떤 종류의 사고와 작업이 필요한지를 먼저 정의해야 합니다. AI fluency는 결국 여러분 자신의 도메인 전문성에서 시작합니다. Platform Awareness는 사용 가능한 AI 시스템의 고유한 강점과 한계를 이해하는 것입니다. 예를 들어 어떤 모델은 속도가 빠르고, 어떤 모델은 깊은 추론에 강합니다. Task Delegation은 이 이해를 바탕으로 실제로 작업을 분배하는 것입니다. 핵심 메시지는 — 모든 것을 AI에게 맡기는 것이 아니라, 각 작업에 가장 적합한 주체를 선택하는 것입니다.

    이제 Delegation을 직접 실습해봅시다. databricks console에 접속하여 Playground를 열고, 모델을 Sonnet 4.5로 선택해주세요. 화면에 있는 프롬프트 템플릿을 사용해서, 여러분의 실제 업무 맥락에 맞게 바꿔서 입력해보세요. 예시로 데이터 분석 에이전트 PoC를 보여드리고 있습니다. 2-3분 정도 시간을 드리겠습니다. 자신의 업무에서 AI에게 위임하고 싶은 작업을 떠올려보시고, Claude와 함께 위임 계획을 세워보세요. 이 실습의 핵심은 — AI에게 무작정 시키는 것이 아니라, 무엇을 맡길지 전략적으로 논의하는 것입니다.

    Description 깊이 보기

    AI와 효과적으로 소통하는 기술 — 단순한 프롬프트 작성이 아니다

    작업을 설명하고, 질문하고, 맥락을 제공하고, 상호작용을 안내한다

    1. Product Description

    무엇을 만들 것인가 —
    원하는 결과물을 명확히 정의한다

    2. Process Description

    어떻게 접근할 것인가 —
    AI의 사고 과정을 안내한다

    3. Performance Description

    어떻게 행동할 것인가 —
    AI의 행동 방식을 정의한다

    이제 4D의 두 번째, Description을 살펴봅시다. Description은 단순히 프롬프트를 잘 쓰는 것이 아닙니다. AI와 함께 생산적인 협업 환경을 구축하는 소통의 기술입니다. 작업을 설명하고, 질문하고, 맥락을 제공하고, 상호작용을 안내하는 것 모두가 포함됩니다. 이를 세 가지 기둥으로 나눠볼 수 있습니다. Product Description — 무엇을 만들 것인지, Process Description — 어떻게 접근할 것인지, Performance Description — AI가 어떻게 행동하길 원하는지.

    Description 3 Pillars

    1. Product Description — 무엇을 원하는가?

    • 원하는 것을 명확히 기술: 맥락, 형식, 대상, 스타일, 제약 조건
    • AI가 추측하지 않도록 — 필요한 정보를 직접 제공한다

    2. Process Description — 어떻게 접근할 것인가?

    • "어떻게"가 "무엇"보다 중요할 수 있다
    • 구체적 데이터, 핵심 작업, 선호하는 순서 등 사고 과정을 안내

    3. Performance Description — 어떻게 행동할 것인가?

    • AI는 맥락에 따라 다르게 행동하는 인터랙티브 시스템
    • 간결하게 vs 자세하게, 도전적으로 vs 지지적으로 — 원하는 행동 방식을 정의
    세 기둥을 하나씩 보겠습니다. Product Description은 원하는 결과물의 특성을 명확히 정의하는 능력입니다. 맥락, 형식, 대상, 스타일, 제약 조건을 구체적으로 알려줘야 AI가 추측하지 않고 정확한 결과를 내놓습니다. Process Description은 AI의 사고 과정을 안내하는 능력입니다. 어떤 데이터를 사용해야 하는지, 핵심 작업은 무엇인지, 어떤 순서로 진행할지를 알려주는 것입니다. 때로는 "무엇"을 원하는지보다 "어떻게" 접근할지가 더 중요합니다. Performance Description은 AI의 행동 방식을 정의하는 능력입니다. AI는 자판기가 아니라 인터랙티브한 파트너입니다. 간결하게 답할지, 자세히 답할지, 비판적으로 접근할지 등을 명시하면 훨씬 나은 결과를 얻을 수 있습니다.

    Description 실습 실습

    Playground에서 부족한 프롬프트를 개선하는 연습

    프롬프트:

    목표: 프롬프트 Description에 대한 연습 경험

Context:
- 다음 항목들이 부족한 프롬프트를 작성해서 개선하는 것을 연습하려고 해
  - Product description
  - Process description
  - Performance description

Desired output:
위의 컨텍스트 항목이 부족한 프롬프트

Process:
1. Claude가 부족한 프롬프트 예시를 작성한다.
2. 내가 부족한 부분을 개선해서 답변한다.
3. Claude가 개선한 부분에 대해서 평가해준다.

    Claude가 만든 부족한 프롬프트를 Product • Process • Performance 관점에서 개선해보세요

    이번에는 Description을 연습해봅시다. 앞서 사용한 Playground에서 이 프롬프트를 입력해주세요. 이 실습은 세 단계로 진행됩니다. 먼저 Claude가 Product, Process, Performance 중 하나 이상이 부족한 프롬프트 예시를 만들어줍니다. 여러분은 그 프롬프트를 보고 부족한 부분을 찾아서 개선합니다. 그러면 Claude가 여러분의 개선 사항에 대해 피드백을 줍니다. 2-3분 정도 시간을 드리겠습니다. 이 실습의 핵심은 — 좋은 프롬프트란 무엇인지를 Product, Process, Performance 세 관점에서 체계적으로 생각해보는 것입니다.

    4D → Vibe Coding

    "

    There's a new kind of coding I call 'vibe coding' where you fully give in to the vibes, embrace exponentials, and forget that the code even exists.

    — Andrej Karpathy, OpenAI 창립 멤버, 2025년 2월

    4D Framework이 Vibe Coding에서 의미하는 것

    Delegation → 코드 작성을 AI에게 위임
    Description → 자연어로 의도를 전달
    Discernment → AI 생성 코드의 품질 검증
    Diligence → 보안, 라이선스, 윤리 확인

    역할 전환: 코더 → 오케스트레이터

    Andrej Karpathy는 OpenAI 창립 멤버이자 전 Tesla AI 디렉터로, 2025년 2월에 Vibe Coding이라는 용어를 처음 사용했습니다. Vibe Coding은 4D Framework과 자연스럽게 연결됩니다. 코드 작성 자체를 AI에게 위임(Delegation)하고, 자연어로 의도를 명확히 기술(Description)하고, 생성된 코드를 분별(Discernment)하여 품질을 검증하고, 보안과 윤리적 측면을 책임(Diligence)지는 것이죠. 결국 개발자의 역할이 코드를 직접 작성하는 코더에서, AI를 지휘하는 오케스트레이터로 전환됩니다. 오늘 핸즈온에서 이 역할 전환을 직접 경험하실 겁니다.

    AI 코딩 도구의 발전

    도구 방식 특징 4D 관점
    Copilot 자동 완성 에디터 내 인라인 제안, Tab으로 수락 부분 위임, 수동 분별
    Cursor 채팅 + 편집 멀티 파일 편집, Composer 모드 대화형 기술, 반자동 분별
    Claude Code 에이전틱 터미널 기반, 자율적 도구 사용, MCP 확장 전면 위임, 체계적 책임

    자동 완성 → 대화형 편집 → 자율적 에이전트

    에이전틱 코딩 = AI가 도구를 사용하여 자율적으로 계획 → 실행 → 검증

    AI 코딩 도구의 발전 단계를 4D 관점에서 살펴봅시다. GitHub Copilot은 자동 완성 방식으로, 개발자가 코드 작성의 일부만 위임하고 수동으로 결과를 분별합니다. Cursor는 대화형으로 의도를 전달(Description)할 수 있게 되었고, 멀티 파일 편집으로 분별이 반자동화되었습니다. Claude Code는 에이전틱 방식으로 전면적인 위임이 가능해졌습니다. AI가 직접 파일을 읽고 쓰고, 명령을 실행하고, 결과를 검증합니다. 다만 전면 위임일수록 Diligence(책임)가 더 중요해집니다 — 보안, 권한, 윤리적 사용에 대한 체계적 관리가 필요합니다. 에이전틱 코딩이란 결국 AI가 도구를 사용하여 자율적으로 계획하고 실행하고 검증하는 방식을 의미합니다.

    Claude의 차별점

    에이전틱 도구 사용

    파일 읽기/쓰기, 명령 실행, 검색까지 자율적으로 수행

    MCP 확장

    Model Context Protocol을 통해 외부 서비스와 연동 — 무한 확장

    안전한 실행 + 한국어

    사용자 승인 기반 실행, 자연스러운 한국어 프롬프트 지원

    Claude가 다른 AI 코딩 도구와 구별되는 핵심 차별점 네 가지입니다. 첫째, 200K 토큰의 긴 컨텍스트 윈도우로 대규모 프로젝트의 코드를 한 번에 이해할 수 있습니다. 둘째, 에이전틱 도구 사용 — 단순히 코드를 제안하는 것이 아니라, 직접 파일을 편집하고 명령을 실행합니다. 셋째, MCP(Model Context Protocol)를 통한 확장성입니다. 외부 서비스, 데이터베이스, API 등과 연동할 수 있어 오늘 핸즈온에서 직접 경험하실 핵심 기능입니다. 넷째, 위험한 작업은 사용자 승인을 요청하는 안전한 실행 환경과 한국어를 자연스럽게 이해하는 능력입니다.

    정리 & Q&A

    핵심 정리

    • AI Fluency — 4D Framework(위임, 기술, 분별, 책임)으로 AI 협업을 체계적으로
    • Vibe Coding — AI에게 의도를 전달하여 코드를 생성하는 새로운 개발 패러다임
    • 에이전틱 코딩 — AI가 도구를 사용하여 자율적으로 계획, 실행, 검증
    • Claude Code — 긴 컨텍스트, MCP 확장, 안전한 에이전틱 코딩 도구

    Claude Code를 직접 설치하고, 핵심 기능을 실습합니다


    질문이 있으신가요?

    세션을 마무리하며 핵심 내용을 정리합니다. AI Fluency는 AI와 효과적으로 협업하는 역량이며, 4D Framework — 위임, 기술, 분별, 책임 — 으로 구체화됩니다. Vibe Coding은 코드를 직접 작성하는 것이 아니라 AI에게 의도를 전달하는 새로운 패러다임이고, 에이전틱 코딩은 AI가 자율적으로 개발 작업을 수행하는 방식이며, Claude Code는 이를 가능하게 하는 강력한 도구라는 것을 기억해주세요. 질문을 받은 후 바로 다음 섹션으로 넘어가겠습니다. 다음 섹션에서는 실제로 Claude Code를 설치하고 사용해봅니다.
