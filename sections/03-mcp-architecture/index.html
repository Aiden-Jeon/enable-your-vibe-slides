    MCP 아키텍처

    Section 03 · Lecture + Hands-on · 25분

    세 번째 섹션에서는 MCP(Model Context Protocol) 아키텍처를 다룹니다. AI 모델이 외부 도구와 소통하는 표준 프로토콜인 MCP의 개념, 구조, 그리고 실제로 FastMCP를 사용해 MCP 서버를 만드는 실습까지 진행합니다. 총 25분, 강의 15분 + 실습 10분 구성입니다.

    학습 목표

    1. MCP(Model Context Protocol)의 개념과 필요성 이해
    2. MCP의 아키텍처와 세 가지 프리미티브 이해
    3. FastMCP로 간단한 MCP 서버를 직접 만들기
    이번 섹션의 학습 목표는 세 가지입니다. 첫째, MCP가 무엇이고 왜 필요한지를 이해합니다. 둘째, MCP의 내부 구조와 핵심 개념인 세 가지 프리미티브를 알아봅니다. 셋째, 실제로 Python FastMCP 라이브러리를 사용해 MCP 서버를 만들어봅니다.

    MCP란?

    Model Context Protocol

    AI 모델이 외부 도구 및 데이터 소스와 소통하는 표준 프로토콜

    • Anthropic이 2024년 11월 공개한 오픈 프로토콜
    • AI 모델과 외부 세계 사이의 "USB-C" 같은 존재
    • 어떤 AI 모델이든, 어떤 도구든 하나의 규격으로 연결
    MCP는 Model Context Protocol의 약자입니다. Anthropic이 2024년 11월에 공개한 오픈 프로토콜로, AI 모델이 외부 도구나 데이터 소스와 표준화된 방식으로 소통할 수 있게 해줍니다. 쉽게 비유하면, 예전에 핸드폰마다 다른 충전 케이블을 쓰다가 USB-C 하나로 통일된 것처럼, AI와 도구 연결을 하나의 규격으로 통일하는 것입니다.

    왜 MCP가 필요한가?

    Before MCP: N x M 문제

    Claude ──── Slack
    Claude ──── GitHub
    Claude ──── DB
    GPT ────── Slack
    GPT ────── GitHub
    GPT ────── DB

    3 모델 x 3 도구 = 9개 통합

    After MCP: N + M 해결

    Claude ─┐
            ├── MCP ──┬── Slack
    GPT ───┤         ├── GitHub
            ├──        └── DB
    Gemini ─┘

    3 모델 + 3 도구 = 6개 통합
    MCP가 왜 필요한지 이해하려면 N x M 문제를 알아야 합니다. MCP가 없으면, 모델마다 각각의 도구에 맞는 통합을 별도로 만들어야 합니다. 모델 3개, 도구 3개면 9개의 통합이 필요하죠. MCP를 사용하면 각 모델은 MCP만 지원하면 되고, 각 도구도 MCP 서버만 만들면 됩니다. 3 + 3 = 6개만 있으면 모든 조합이 가능해집니다. 도구와 모델이 늘어날수록 이 효과는 더 커집니다.

    MCP 아키텍처

    ┌─────────────────────────────────────────────────────────┐
    │                      Host (Claude Code)                      │
    │                                                          │
    │  ┌──────────┐   ┌──────────┐   ┌──────────┐       │
    │  │ MCP      │   │ MCP      │   │ MCP      │       │
    │  │ Client A │   │ Client B │   │ Client C │       │
    │  └────┬─────┘   └────┬─────┘   └────┬─────┘       │
    └───────┼──────────────┼──────────────┼────────────────┘
            │              │              │
            ▼              ▼              ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ MCP      │   │ MCP      │   │ MCP      │
    │ Server A │   │ Server B │   │ Server C │
    └──┬───┬───┘   └──┬───┬───┘   └──┬───┬───┘
       │   │         │   │         │   │
       ▼   ▼         ▼   ▼         ▼   ▼
    Tools Resources  Tools Resources  Tools Resources
    MCP 아키텍처는 세 가지 레이어로 구성됩니다. Host는 Claude Code 같은 AI 애플리케이션입니다. Host 안에는 여러 MCP Client가 있고, 각 Client는 하나의 MCP Server와 1:1로 연결됩니다. MCP Server는 실제 도구(Tools)와 리소스(Resources)를 제공합니다. Client와 Server 사이의 통신은 JSON-RPC 2.0 프로토콜을 사용합니다. 이 구조 덕분에 Host는 여러 서버를 동시에 사용할 수 있습니다.

    세 가지 프리미티브

    프리미티브 설명 비유
    Tools 모델이 호출할 수 있는 함수 API 엔드포인트
    Resources 모델이 읽을 수 있는 데이터 GET 엔드포인트
    Prompts 미리 정의된 프롬프트 템플릿 API 문서 / 가이드

    오늘 실습에서는 Tools에 집중합니다!

    MCP는 세 가지 핵심 프리미티브를 제공합니다. 첫째, Tools는 모델이 호출할 수 있는 함수입니다. 계산기, 파일 읽기, API 호출 등의 액션을 수행합니다. 둘째, Resources는 모델이 읽을 수 있는 데이터입니다. 파일 내용, 데이터베이스 쿼리 결과 등을 제공합니다. 셋째, Prompts는 미리 정의된 프롬프트 템플릿입니다. 오늘 실습에서는 가장 많이 사용되는 Tools에 집중하겠습니다.

    Tools 상세

    MCP Tool은 세 가지 요소로 구성됩니다:

    구성 요소

    • 이름 (name) - 도구 식별자
    • 설명 (description) - AI가 도구를 이해하는 핵심
    • 입력 스키마 (input schema) - JSON Schema 형태의 파라미터 정의

    JSON Schema 예시

    {
  "name": "echo",
  "description": "메시지를 그대로 반환",
  "inputSchema": {
    "type": "object",
    "properties": {
      "message": {
        "type": "string",
        "description": "반환할 메시지"
      }
    },
    "required": ["message"]
  }
}
    Tools를 좀 더 자세히 알아봅시다. MCP Tool은 이름, 설명, 입력 스키마 세 가지로 구성됩니다. 특히 description이 중요한데, AI 모델이 이 설명을 보고 어떤 도구를 언제 사용할지 판단하기 때문입니다. 입력 스키마는 JSON Schema 형태로 파라미터를 정의합니다. 오른쪽 예시처럼 echo 도구는 message라는 문자열 파라미터를 받습니다.

    FastMCP 소개

    FastMCP: Python으로 MCP 서버를 가장 쉽게 만드는 방법

    특징

    • 데코레이터 기반의 직관적 API
    • 타입 힌트에서 자동으로 스키마 생성
    • docstring이 도구 설명이 됨
    • uv로 의존성 없이 바로 실행

    설치

    # uv를 사용한 설치 (권장)
uv add fastmcp

# pip를 사용한 설치
pip install fastmcp
    FastMCP는 Python으로 MCP 서버를 가장 쉽게 만들 수 있는 라이브러리입니다. FastAPI에서 영감을 받아, 데코레이터 기반의 직관적인 API를 제공합니다. Python 타입 힌트에서 자동으로 JSON Schema를 생성하고, 함수의 docstring이 도구의 설명으로 자동 변환됩니다. uv를 사용하면 가상환경 설정 없이 바로 실행할 수 있어 매우 편리합니다.

    코드 예제: Hello MCP 서버

    """Exercise 01: Hello MCP - 첫 번째 MCP 서버"""
from fastmcp import FastMCP

mcp = FastMCP("Hello MCP")           # 서버 인스턴스 생성

@mcp.tool()                           # 도구로 등록
def echo(message: str) -> str:
    """메시지를 그대로 반환합니다."""
    return f"Echo: {message}"

@mcp.tool()                           # 두 번째 도구
def greet(name: str) -> str:
    """이름을 받아 인사합니다."""
    return f"안녕하세요, {name}님!"

if __name__ == "__main__":
    mcp.run()                         # 서버 실행

    단 17줄로 MCP 서버 완성!

    첫 번째 코드 예제입니다. FastMCP로 MCP 서버를 만드는 기본 패턴을 봅시다. 하이라이트된 줄을 따라가면: 먼저 FastMCP를 import하고, 서버 인스턴스를 생성합니다. 이름은 "Hello MCP"입니다. @mcp.tool() 데코레이터로 함수를 MCP 도구로 등록합니다. echo 도구는 메시지를 그대로 반환하고, greet 도구는 이름을 받아 인사를 합니다. 마지막으로 mcp.run()으로 서버를 실행합니다. 단 17줄만으로 2개의 도구를 가진 MCP 서버가 완성됩니다!

    코드 예제: 계산기 MCP 서버

    """Exercise 02: Calculator MCP - 계산기 MCP 서버"""
from fastmcp import FastMCP

mcp = FastMCP("Calculator")

@mcp.tool()
def add(a: float, b: float) -> float:
    """두 수를 더합니다."""
    return a + b

@mcp.tool()
def subtract(a: float, b: float) -> float:
    """두 수를 뺍니다."""
    return a - b

@mcp.tool()
def divide(a: float, b: float) -> float:
    """두 수를 나눕니다."""
    if b == 0:
        raise ValueError("0으로 나눌 수 없습니다.")
    return a / b

    핵심: 타입 힌트가 파라미터 스키마를, docstring이 도구 설명을 자동 생성!

    두 번째 예제는 계산기 MCP 서버입니다. 구조는 첫 번째와 동일하지만, 여러 도구를 등록하는 패턴을 보여줍니다. add, subtract, divide 세 가지 도구를 등록합니다. divide 도구에서는 0으로 나누기 에러 처리도 보여줍니다. 핵심 포인트는 타입 힌트 float가 자동으로 JSON Schema의 number 타입이 되고, docstring이 도구의 설명으로 자동 변환된다는 것입니다. FastMCP가 이 모든 것을 자동으로 처리해줍니다.

    Claude Code에서 MCP 서버 연결

    .claude/settings.local.json 파일에 설정을 추가합니다:

    {
  "mcpServers": {
    "hello-mcp": {
      "command": "uv",
      "args": [
        "run",
        "--with", "fastmcp",
        "fastmcp", "run",
        "exercise_01_hello_mcp.py"
      ]
    },
    "calculator": {
      "command": "uv",
      "args": [
        "run",
        "--with", "fastmcp",
        "fastmcp", "run",
        "exercise_02_calculator_mcp.py"
      ]
    }
  }
}

    uv run --with fastmcp: 별도 설치 없이 바로 실행!

    MCP 서버를 만들었으면 Claude Code에 연결해야 합니다. 프로젝트 루트의 .claude/settings.local.json 파일에 mcpServers 섹션을 추가합니다. command는 uv를 사용하고, args에서 --with fastmcp로 의존성을 지정합니다. fastmcp run 명령으로 서버 파일을 실행합니다. uv run --with 패턴을 사용하면 별도의 패키지 설치 없이 바로 FastMCP 서버를 실행할 수 있어서 매우 편리합니다. 설정 후 Claude Code를 재시작하면 도구를 사용할 수 있습니다.
