Skills Ecosystem

Skills 생태계: Claude Code를 팀의 워크플로우에 맞게 확장하기

Section 04 | 15분 | 강의

이번 섹션에서는 Claude Code의 확장 체계인 Skills 생태계에 대해 알아봅니다. MCP가 외부 도구 연결이라면, Skills는 Claude Code 자체의 행동 패턴을 정의하는 방법입니다. 강의 전용 섹션으로, 15분 동안 개념을 익히고 다음 실습 섹션에서 직접 활용하게 됩니다.

학습 목표

  • Claude Code의 Skills와 Agents 개념 이해
  • SKILL.md 파일 구조와 작성법 학습
  • Skills 생태계를 활용한 워크플로우 표준화 가능성 파악
세 가지 학습 목표가 있습니다. 첫째, Skills와 Agents가 무엇인지 정확히 이해합니다. 둘째, SKILL.md 파일을 직접 작성할 수 있을 정도로 구조를 파악합니다. 셋째, 이 개념들이 팀 단위에서 어떻게 워크플로우를 표준화할 수 있는지 가능성을 봅니다.

Claude Code의 확장 체계

Claude Code를 커스터마이징하는 세 가지 방법

┌─────────────────────────────────────────────┐
│            Claude Code 확장 체계              │
├───────────────┬──────────────┬──────────────┤
│    Skills     │    Agents    │    Hooks     │
│               │              │              │
│  /명령어 호출  │  @에이전트    │  이벤트 트리거 │
│  SKILL.md     │  agent.md    │  hooks 설정   │
│  워크플로우    │  전문 역할    │  자동화       │
└───────────────┴──────────────┴──────────────┘
Claude Code를 확장하는 방법은 크게 세 가지입니다. Skills: 슬래시 명령어로 호출하는 커스텀 워크플로우. Agents: 특정 역할에 특화된 AI 에이전트. Hooks: 특정 이벤트에 자동으로 트리거되는 스크립트. 이번 섹션에서는 Skills와 Agents를 중심으로 다루고, Hooks는 간단히 언급하겠습니다.

Skills란?

  • SKILL.md 파일로 정의하는 커스텀 슬래시 커맨드
  • 프로젝트 내 .claude/skills/ 디렉토리에 위치
  • Claude Code가 특정 작업을 일관된 방식으로 수행하도록 가이드
  • 호출 방법: /skill-name 또는 자동 인식
핵심 아이디어: "이 작업은 항상 이 절차대로 해줘"를 코드화한 것
Skills의 핵심 개념을 설명합니다. SKILL.md는 마크다운 파일로, Claude Code에게 특정 작업의 수행 절차를 알려줍니다. 예를 들어 "슬라이드를 만들어줘"라고 할 때, 매번 스타일 가이드와 구조를 설명하는 대신, SKILL.md에 한번 정의해두면 일관된 결과물을 얻을 수 있습니다. 이것은 팀 전체가 동일한 품질의 결과물을 만들 수 있게 해주는 강력한 기능입니다.

SKILL.md 파일 구조


# 스킬-이름 스킬

스킬에 대한 한줄 설명

## 사용법
/skill-name [파라미터1] [파라미터2]

## 입력
- **파라미터1**: 설명
- **파라미터2**: 설명

## 처리 절차
1. 레퍼런스 파일을 읽어 컨벤션 확인
2. 입력을 분석하여 구성 결정
3. 결과물 생성
4. 지정 경로에 저장

## 출력
- 생성되는 파일 경로와 설명
SKILL.md의 기본 구조입니다. 크게 네 부분으로 나뉩니다. 첫째, 스킬 이름과 설명 - Claude Code가 이 스킬이 무엇을 하는지 파악합니다. 둘째, 사용법과 입력 파라미터 - 어떤 정보가 필요한지 정의합니다. 셋째, 처리 절차 - 가장 중요한 부분으로, 단계별 수행 절차를 기술합니다. 넷째, 출력 - 어떤 파일이 어디에 생성되는지 명시합니다. 이 구조를 따르면 Claude Code가 안정적으로 작업을 수행합니다.

실제 예시: slide-deck 스킬

이 프로젝트에서 실제 사용 중인 스킬


# slide-deck 스킬

reveal.js 기반 슬라이드 프레젠테이션을 생성합니다.

## 사용법
/slide-deck [섹션번호] [주제개요]

## 처리 절차
1. references/slide-conventions.md를 읽어
   슬라이드 스타일 가이드 확인
2. templates/reveal-template.html을 기반으로
   슬라이드 생성
3. 주제개요를 분석하여 적절한
   슬라이드 수와 구성 결정
4. 각 슬라이드에 스피커 노트 포함
5. sections/[번호]-[이름]/slides/index.html에 저장

## 출력
- sections/XX-name/slides/index.html
이 프레젠테이션 자체가 slide-deck 스킬로 만들어진 것입니다. 하이라이트되는 부분을 따라가 보겠습니다. 먼저 스킬 이름과 설명이 있고, 사용법에서 섹션번호와 주제개요를 입력받습니다. 처리 절차가 핵심인데, 레퍼런스 파일을 먼저 읽고, 템플릿 기반으로 생성하며, 스피커 노트를 포함하는 등 구체적인 지시를 담고 있습니다. 이렇게 정의하면 누가 실행해도 동일한 스타일의 슬라이드가 생성됩니다.

레퍼런스 파일

Skills가 참조하는 상세 가이드라인

.claude/skills/slide-deck/
├── SKILL.md                        ← 스킬 정의
└── references/
    └── slide-conventions.md        ← 상세 스타일 가이드

     ┌──────────────────────────┐
     │  slide-conventions.md    │
     │                          │
     │  - 테마 색상 (#1a1a2e)   │
     │  - 레이아웃 원칙          │
     │  - 슬라이드 유형별 패턴   │
     │  - 코드 하이라이팅 규칙   │
     └──────────────────────────┘
SKILL.md는 큰 그림을 정의하고, references 디렉토리의 파일들이 세부 사항을 담당합니다. 예를 들어 slide-deck 스킬은 slide-conventions.md를 참조합니다. 여기에는 테마 색상 코드, 레이아웃 원칙, 슬라이드 유형별 HTML 패턴 등 매우 구체적인 가이드라인이 들어있습니다. 이렇게 분리하면 스킬의 처리 절차는 바꾸지 않으면서 스타일 가이드만 업데이트할 수 있습니다.

Custom Agents

  • .claude/agents/ 디렉토리에 마크다운 파일로 정의
  • 특정 역할에 특화된 AI 에이전트
  • 도구 사용 권한과 제약 조건을 명시
  • 호출 방법: @agent-name
핵심 아이디어: "이 역할은 이 범위 안에서만 일해줘"를 정의한 것
Agents는 Skills와는 다른 개념입니다. Skills가 작업의 절차를 정의한다면, Agents는 역할과 권한을 정의합니다. 마크다운 파일 하나로 에이전트의 역할, 사용 가능한 도구, 제약 조건을 명시합니다. 예를 들어 읽기 전용 에이전트는 파일을 탐색할 수 있지만 수정은 할 수 없도록 제한할 수 있습니다.

Agent 예시

@researcher

읽기 전용 리서치 에이전트

  • 문서, API, 코드 탐색
  • 기술 비교 및 분석
  • 파일 수정 불가
  • 결과: 보고서 형태

@researcher Databricks Genie API의
최신 엔드포인트를 조사해줘

@validator

코드/구조 검증 에이전트

  • 파일 구조 검증
  • 코드 구문 검사
  • 파일 수정 불가
  • 결과: 검증 보고서

@validator sections/03-mcp-architecture/
섹션을 검증해줘
이 프로젝트에서 사용 중인 두 에이전트를 비교해 보겠습니다. researcher는 정보 수집 전문입니다. 최신 API 문서를 조사하거나 기술 비교를 할 때 사용합니다. validator는 품질 검증 전문입니다. 생성된 파일의 구조, 코드 문법, 교차 참조를 확인합니다. 두 에이전트 모두 읽기 전용이라는 공통점이 있습니다. 이렇게 권한을 제한하면 안전하게 작업을 분담할 수 있습니다.

Skills vs Agents

Skills Agents
정의 작업 절차 (워크플로우) 역할과 권한 (페르소나)
파일 SKILL.md agent-name.md
위치 .claude/skills/ .claude/agents/
호출 /skill-name @agent-name
초점 "무엇을, 어떻게 할 것인가" "누가, 어떤 범위에서 할 것인가"
Skills와 Agents의 핵심 차이를 정리한 표입니다. Skills는 "무엇을 어떻게 할 것인가"에 초점을 맞춥니다. 슬래시 명령어로 호출합니다. Agents는 "누가 어떤 범위에서 할 것인가"에 초점을 맞춥니다. 골뱅이로 호출합니다. Skills는 절차적이고, Agents는 역할 기반입니다. 둘 다 마크다운 파일로 정의되며, .claude 디렉토리 아래에 위치합니다. 실무에서는 이 둘을 조합해서 사용합니다.

이 프로젝트의 Skills 생태계

3개 Skills + 2개 Agents로 교육 자료 제작 자동화

.claude/
├── skills/
│   ├── slide-deck/         ← /slide-deck
│   │   ├── SKILL.md           슬라이드 생성
│   │   └── references/
│   ├── hands-on-code/      ← /hands-on-code
│   │   ├── SKILL.md           실습 코드 생성
│   │   └── references/
│   └── section-builder/    ← /section-builder
│       ├── SKILL.md           섹션 통합 빌드
│       └── references/
└── agents/
    ├── researcher.md       ← @researcher
    │                          정보 조사
    └── validator.md        ← @validator
                               품질 검증
지금 보고 계신 이 프레젠테이션 자체가 이 Skills 생태계로 만들어진 것입니다. slide-deck 스킬은 reveal.js 슬라이드를 생성합니다. hands-on-code 스킬은 실습용 Python 코드를 생성합니다. section-builder 스킬은 이 둘을 통합하여 완전한 섹션을 만듭니다. researcher 에이전트는 최신 기술 정보를 조사합니다. validator 에이전트는 생성된 결과물의 품질을 검증합니다. 이렇게 5개의 구성요소가 협력하여 9개 섹션의 교육 자료를 만들고 있습니다.

Skills 생태계의 가능성

팀 워크플로우 표준화

  • 코드 리뷰 표준화: 팀의 코딩 컨벤션을 SKILL.md에 정의
    /code-review - 보안, 성능, 스타일을 일관되게 검사
  • 문서화 자동화: API 문서, 변경 로그 생성 표준화
    /generate-docs - 항상 같은 형식의 문서 생성
  • 테스트 생성: 팀의 테스트 패턴을 스킬로 정의
    /write-tests - 팀 표준에 맞는 테스트 자동 생성
  • 온보딩: 신규 팀원이 팀의 워크플로우를 즉시 활용
    Git clone 후 바로 팀의 Skills 사용 가능
Skills 생태계의 진짜 힘은 팀 단위 표준화에 있습니다. 코드 리뷰를 예로 들면, 팀마다 다른 리뷰 기준이 있을 텐데, 이를 SKILL.md에 정의하면 누가 리뷰를 하든 동일한 기준이 적용됩니다. 문서화, 테스트 생성도 마찬가지입니다. 특히 온보딩에서 강력합니다. 신규 팀원이 프로젝트를 클론하면 .claude 디렉토리에 있는 모든 Skills를 바로 사용할 수 있습니다. 팀의 노하우가 코드와 함께 버전 관리되는 것입니다.

정리

  • Skills: SKILL.md로 정의하는 커스텀 슬래시 커맨드
    작업 절차를 코드화하여 일관된 결과물 보장
  • Agents: 특정 역할에 특화된 AI 에이전트
    역할과 권한을 명시하여 안전한 작업 분담
  • 생태계: Skills + Agents + Hooks = 팀 워크플로우 표준화
    Git으로 관리되어 팀 전체가 공유
다음 섹션 예고: Section 06에서 Skills를 직접 만들어 봅니다!
정리하겠습니다. Skills는 작업 절차를 SKILL.md에 정의하여 슬래시 명령어로 호출합니다. Agents는 역할과 권한을 정의하여 골뱅이로 호출합니다. 이 둘과 Hooks를 합쳐 Skills 생태계라고 부릅니다. 모두 .claude 디렉토리 아래 마크다운 파일이므로, Git으로 버전 관리할 수 있습니다. Section 06에서는 직접 Skills를 작성하는 실습을 진행합니다.
