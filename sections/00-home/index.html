    환경 설정

    아래 3단계로 실습 환경을 준비합니다 

    # 1. 프로젝트 클론
git clone https://github.com/Aiden-Jeon/enable-your-vibe-exercise
cd enable-your-vibe-exercise
    # 2. 패키지 설치 (모든 실습 의존성 한 번에)
uv sync
    # 3. 설치 확인
uv run python -c "import fastmcp; print('Ready!')"
    실제로 만들기 위해 우선 환경 설정을 해야 됩니다. 코드를 남겨놨으니 참석하신 분들은 프로젝트 클론하시고 패키지 설치해 주시면 됩니다. 여기서 uv sync라는 명령어를 사용하는데, uv가 무엇인지는 다음 슬라이드에서 설명드리겠습니다. 설치 확인이 됐는지 한 번 확인해 보겠습니다.

    uv란?

    Rust로 작성된 초고속 Python 패키지 매니저

    pip 대비 10-100배 빠른 설치 속도 — 가상환경 + 패키지 관리 + 스크립트 실행을 하나로 통합

    pip uv
    pip install -r requirements.txt uv sync
    pip install <pkg> uv add <pkg>
    python script.py uv run python script.py
    앞에서 uv sync를 사용했는데, uv가 무엇인지 간단히 설명드리겠습니다. uv는 Rust로 작성된 초고속 Python 패키지 매니저입니다. pip 대비 10배에서 100배까지 빠른 설치 속도를 보여줍니다. 기존에 pip install -r requirements.txt 하던 것을 uv sync 한 줄로 대체할 수 있고, pip install 대신 uv add, python script.py 대신 uv run python script.py를 사용합니다. 가상환경 생성, 패키지 관리, 스크립트 실행을 하나의 도구로 통합해서 관리할 수 있다는 것이 큰 장점입니다.

    uvx — 설치 없이 실행

    Python 도구를 설치 없이 바로 실행 — Node.js의 npx와 유사 

    # MCP 서버를 uvx로 바로 실행
uvx fastmcp run server.py

    MCP 설정에서의 활용 패턴:

    {
  "mcpServers": {
    "my-server": {
      "command": "uvx",
      "args": ["fastmcp", "run", "server.py"]
    }
  }
}
    특히 중요한 것이 uvx입니다. uvx는 Python 도구를 설치 없이 바로 실행할 수 있게 해줍니다. Node.js의 npx와 유사한 개념입니다. MCP 서버를 여러 프로젝트에서 사용할 때 uvx로 경로를 설정하면 바로 실행되는 장점이 있습니다. 실제로 MCP 설정 파일에서 command를 uvx로 지정하는 패턴을 많이 사용합니다. uvx를 통해서 로컬 MCP 서버들을 관리하는 경우가 많이 있습니다. 이후 섹션 03에서 MCP를 배우고, 섹션 04에서 Genie MCP를 만들 때 이 패턴을 직접 사용하게 됩니다.
