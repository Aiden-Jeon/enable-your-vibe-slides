    오늘 세션의 세 가지 목표입니다. 첫 번째로, 단순히 Vibe Coding만 설명하는 것이 아니라 Anthropic에서 AI Fluency라고 하는 개념을 소개했습니다. AI Fluency는 우리가 어떻게 AI 툴들과 협업할 건지에 대한 개관적인 내용이라고 봐주시면 됩니다. 두 번째로, 여러 가지 도구들이 있는데 Claude Code가 제일 강력하고 사용할 수 있는 도구들이 많이 나오고 있기 때문에 오늘 세션에서는 Claude Code 사용법을 설명드리겠습니다. MCP나 Skills 같은 것들을 사용해야 좀 더 잘 쓸 수 있기 때문에 이런 것들을 하나씩 구현해서 어떻게 동작하는지 이해할 겁니다. 마지막으로, 회사에서 이러한 도구들을 다 모은 게 AI Dev Kit이라고 하는 것이 있습니다. 이에 대한 내용을 간단하게 설명드리겠습니다.

    무엇을 만드나요?

    Genie Space를 만드는 도구를 생성하고, Claude Code로 구현하는 과정을 경험합니다

    이 모든 것을 Vibe Coding으로 — 코드를 직접 작성하지 않고, AI에게 의도를 전달하여 구현합니다

    저희가 만들 것은 Genie Space를 만들 수 있는 MCP 서버입니다. 이것을 직접 구현하는 것이 아니라 전부 다 Claude Code한테 시켜서 만들도록 해보겠습니다.

    환경 설정

    아래 3단계로 실습 환경을 준비합니다 

    # 1. 프로젝트 클론
git clone https://github.com/Aiden-Jeon/enable-your-vibe-exercise
cd enable-your-vibe-exercise
    # 2. 패키지 설치 (모든 실습 의존성 한 번에)
uv sync
    # 3. 설치 확인
uv run python -c "import fastmcp; print('Ready!')"

    uv sync 한 번으로 모든 실습 패키지가 설치됩니다 — 섹션별 pip install은 필요 없습니다!

    실제로 만들기 위해 우선 환경 설정을 해야 됩니다. 코드를 남겨놨으니 참석하신 분들은 프로젝트 클론하시고 패키지 설치해 주시면 됩니다. 패키지로는 UV를 사용할 건데, uv sync를 하면 적어 놓은 패키지들이 다 설치됩니다. MCP가 나오면서 UV를 많이 사용하게 되는데, UV는 결국 파이썬 가상 환경을 관리해주는 패키지라고 봐주시면 됩니다. Rust로 작성되어 있어서 매우 빠르고요. 특히 UVX라고 하는 프로젝트가 있는데, MCP 서버를 여러 프로젝트에서 사용할 때 UVX로 경로를 설정하면 바로 실행되는 장점이 있습니다. UVX를 통해서 로컬 MCP 서버들을 관리하는 경우가 많이 있습니다. 설치 확인이 됐는지 한 번 확인해 보겠습니다.
